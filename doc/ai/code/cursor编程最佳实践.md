## Plan先行

对于稍微大一些的任务, 遵循计划先行的思想, 使用Plan模式, 提示词模版如下:

```txt
任务如下:
1. 要点1
2. 要点2
3. 要点3

要求如下:
1. 尽可能地使用测试驱动开发的编程范式, 对于适合做单元测试的需求, 执行编程之前需要先写单元测试, 然后再写代码, 修改代码直到单测通过

请按照上面的要求生成详细的计划, 如果有疑惑请提出来, 如果我的方案有不合理的地方, 请质疑我.

```

然后仔细审核给出的plan文档, 如果发现有问题, 可以要求修改

期间还可以进行像Ask模式一样的讨论

最后经过不断打磨, plan文档符合你的要求之后, 可以要求cursor执行, 提示词模版如下:

```txt
请严格按照最新的plan文档执行, 如果遇到问题请停下来, 并说明情况

```
